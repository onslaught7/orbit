# System Architecture and Design Decisions

## Functional Requirements (What does the system do ?).

- User Registers (with an anonymous reveal name)
- Gets Matches and Recommendations based on personal data, choices and defined proximity
- Proximity is mapped throughout the day and best matches are shown (realtime or after a buffer)
- User connects with a request
- Successful connection results in the unlocking of a chat interface among the connections
- Users can request for groups among there connections (on successful acceptance)
- Users can reveal their other socials on the platform (Private, Public, To Connections only, to connections of connections only)
